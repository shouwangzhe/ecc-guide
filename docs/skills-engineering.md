# 架构与工程技能

## API 设计

| 技能 | 作用 |
| --- | --- |
| api-design | REST API 设计模式，包括资源命名、版本控制、错误响应和 OpenAPI 规范 |
| backend-patterns | 后端架构模式、API 设计、数据库访问层和服务层设计 |
| mcp-server-patterns | 使用 Node/TypeScript SDK 构建 MCP 服务器，包括工具定义、资源和提示 |

## 数据库

| 技能 | 作用 |
| --- | --- |
| postgres-patterns | PostgreSQL 查询优化、索引策略、分区和 Supabase 最佳实践 |
| database-migrations | 数据库迁移最佳实践，包括模式变更、回滚策略和零停机迁移 |
| clickhouse-io | ClickHouse 数据库模式、查询优化、物化视图和大规模数据分析 |
| jpa-patterns | JPA/Hibernate 实体设计、关系映射、查询优化和事务管理 |

## 基础设施与部署

| 技能 | 作用 |
| --- | --- |
| docker-patterns | Docker 和 Docker Compose 模式，用于本地开发和生产部署 |
| deployment-patterns | 部署工作流、CI/CD 流水线模式、Docker 和云部署策略 |

## 安全

| 技能 | 作用 |
| --- | --- |
| security-review | 安全审查技能，用于处理用户输入、认证、API 端点和敏感数据时触发 |
| security-scan | 扫描 Claude Code 配置（.claude/ 目录）中的安全问题和错误配置 |

## 测试

| 技能 | 作用 |
| --- | --- |
| tdd-workflow | TDD 工作流技能，写新功能、修 bug 或重构时使用，强制先写测试 |
| e2e-testing | Playwright E2E 测试模式、Page Object Model 和测试稳定性最佳实践 |
| verification-loop | 全面的验证系统，覆盖构建、类型检查、lint、测试和部署就绪检查 |
| ai-regression-testing | AI 辅助开发的回归测试策略，防止 AI 生成代码引入回归 |

## 代码质量与标准

| 技能 | 作用 |
| --- | --- |
| coding-standards | 通用编码标准、最佳实践和跨语言模式 |
| architecture-decision-records | 在 Claude Code 会话中捕获架构决策，生成 ADR 文档 |
| codebase-onboarding | 分析陌生代码库并生成结构化的入门指南 |
| search-first | 编码前先搜索的工作流，查找现有解决方案避免重复造轮子 |
| blueprint | 将一行目标转化为分步骤的可执行构建计划 |
| plankton-code-quality | 使用 Plankton 在写代码时实时执行代码质量规范 |
| content-hash-cache-pattern | 使用 SHA-256 哈希缓存昂贵的文件处理结果，避免重复计算 |
| project-guidelines-example | 基于实际项目的项目特定技能模板示例 |

## 文档

| 技能 | 作用 |
| --- | --- |
| documentation-lookup | 通过 Context7 MCP 获取最新的库和框架文档 |
| rules-distill | 扫描技能以提取跨领域原则并将其提炼为可复用规则 |
